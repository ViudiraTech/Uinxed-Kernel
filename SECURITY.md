# Security Policy

## Supported Versions

Uinxed is currently under active development, Security updates are provided for recent stable releases:

|   Version   |     Supported      |
| -------     | ------------------ |
| old_version | :x:                |
| ≥ 0.1.1     | :white_check_mark: |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub Issues.
If you discover a security vulnerability in Uinxed, please report it privatelythrough GitHub Security Advisories.
When reporting a vulnerability, please include:

- Affected Uinxed version
- CPU architecture (for example x86_64)
- Affected component (kernel, driver, filesystem, userspace, etc.)
- Description of the vulnerability
- Steps to reproduce the issue
- Potential security impact
- Any available logs, crash information, or proof-of-concept code

## Response Process

After receiving a vulnerability report:

1. The report will be reviewed and confirmed.
2. The maintainers will investigate the issue and determine the impact.
3. If the vulnerability is confirmed, a fix will be developed.
4. A security update or patch will be released.
5. A security advisory may be published after the issue has been fixed.

## Disclosure Policy

Security vulnerabilities should not be publicly disclosed before a fix isavailable.
After a vulnerability has been fixed, relevant details may be publishedthrough a security advisory.

## Security Updates

Security fixes will be included in future Uinxed releases and documented inthe release notes.
